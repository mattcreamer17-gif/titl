A read on where AI actually fits in Titl's stack.

Titl closed $2.5M in January from Cofounders Capital and FIT Ventures. The plan is Florida to twenty states by the end of 2026. The pipeline runs Google Gemini and OpenAI in parallel on every document, stored on Polymesh, priced at $299 a report. Under 5 minutes. 99.5% accuracy. Known.

60Code and database
30Rule-based logic
10Genuine AI reasoning

The question is which parts of the pipeline actually need an LLM, and which parts are traditional code, rules, or human QA wearing an AI hat.

Most teams in this position put LLMs where Postgres would do the job faster, cheaper, and with better guarantees. The 60/30/10 read is that 60% of Titl's work is code and database plumbing (county ingestion, schema normalization, API surface for lenders), 30% is rule-based logic (state statutes, ALTA best practice, lien release patterns), and 10% is genuine AI reasoning (handwriting OCR on messy stamps, ambiguous chain-of-title, fraud pattern detection). Getting that split right is the difference between a margin-healthy seed and a second round that has to cover inference bills.

TitlAlert has a separate shape. Watch, classify, escalate, notify, across thousands of parcels. At ten people, that system either gets a deliberate orchestration layer now or gets rebuilt at Series A for three times the cost.

We built a version of this conversation with Feeld. CTO-led, product company, trying to operationalize AI without hiring a head of AI. A scoped sprint. A workshop, a set of advisory calls, an Organizational Context Architecture, and a Strategic Operations Framework that their engineering team could run against for the next year.

The methodology behind it is Interpretable Context Methodology (ICM), submitted to ACM TiiS. Folder structure as agent architecture. It gives Ray and the team a vocabulary and a structure for the ingestion pipeline that holds up from one state to twenty.

Eduba partners with NLP Logix for work that sits below the orchestration layer. NLP Logix has been in machine learning since 2011 and runs over 150 data scientists.
